### A. Серийная группировка кондиционеров из прайса (Dantex-кейс)
Сегодня RK-07SCDG и RK-09SCDG сгенерились ОТДЕЛЬНЫМИ карточками — excel-путь
не группирует (позиция прайса = карточка), серийность есть только у каталожного
пути (group_by_series + member_prices в подписи). Владелец подтвердил: прайс —
полноценный источник, кондиционеры из него генерить СЕРИЯМИ (одна карточка,
мощности/цены линейкой в подписи), анти-дубль по ключам защитит от пересечений
с каталогом. План: (1) извлечение BTU/серии из наименований прайса (паттерны
«RK-07SCDG…07 BTU», по аналогии с catalog/series.py); (2) группировка выбранных
позиций перед постановкой в excel_items (ключ серии вместо ключа позиции,
члены серии в новой колонке/таблице); (3) preview-подпись с линейкой цен
(переиспользовать render_caption member_prices стиль); (4) research/card один
на серию. Затрагивает: ingest/excel_price.py, orchestrator/excel_pipeline.py,
excel_run.py, bot make/pick/task-пути.

### B. /task с расписанием и автосписком
Владелец хочет: тап по категории → бот САМ присылает нумерованный список из
прайсов (как /find) → выбор номеров (как /pick) → время выгрузки («завтра 9:00»,
count) → генерация к сроку. По сути слияние /task-визарда с auto_tasks-слотами
(orchestrator/auto.py уже умеет слоты по времени) + /find/pick. План: (1) шаг
категории в визарде → кнопки из известных категорий прайса (разделы) + список
топ-N позиций с номерами; (2) шаг времени (парсер «завтра 9:00» есть в
commands.py /plan-пути); (3) постановка в слоты (Slot) вместо немедленного
add_items; (4) excel_run уважает слоты (сейчас тикает всё new сразу).
