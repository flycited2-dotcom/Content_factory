# Spec — UX-волна бота: цена-плашка, ввод команд, опросник заказа (2026-07-05)

Три правки по запросу владельца (скриншоты канала + бота). Дизайн одобрен устно,
реализация — по TDD (см. коммит этой ветки). Ниже — что и зачем, для истории.

## Задача 1 — цена отдельной плашкой-цитатой

**Было:** строка цены `💰 10 672 ₽` внутри подписи.
**Стало:** `<blockquote>💎 <b>10 672 ₽</b></blockquote>` — отдельная плашка-цитата,
жирный номинал, маркер 💎 (выбор владельца вместо «мешочка»).

- Два билдера подписи: `orchestrator/excel_run.py::build_preview_caption` (карточки
  прайса) и `content/render.py::_header`/`_series_header` (B2B-канал климата).
- `parse_mode=HTML` уже был. В excel — `html.escape(name)` (не трогаем, `<артикул>`
  экранируется). В плашку идёт только `_money(...)` → спецсимволов нет, безопасно.
- Тесты: `test_excel_pipeline`, `test_render` обновлены под новый формат.

## Задача 2 — ввод аргументов команд через ForceReply

**Было:** тап пустой команды из меню (`/find`, `/make`, `/pick`) сразу отправлял её
без аргумента — бот отвечал «что ищем?».
**Стало:** пустая команда → приглашение с активным полем ввода (`ForceReply` +
плейсхолдер). Следующий текст владельца = аргумент, дальше обычная обработка.

- `bot/cmd_input.py`: `ARG_COMMANDS`, `bare_arg_command`, `prompt_for`,
  `resolve_reply`, `PendingCmdStore` (SQLite, одноразовый `take`).
- Обвязка в `bot/run.py` (owner-зона). `/excel`, `/status`, `/task` — без изменений.
- Набрал вместо ответа другую команду → отложенная сбрасывается.

## Задача 3 — опросник заказа + лиды в отдельный чат

**Было:** кнопка «Заказать» → `/start ord_<code>` сразу создавала лид владельцу
в тот же управляющий бот («Никита»). Клиенты попадали в бот владельца.
**Стало:** клиент проходит опросник (кол-во → комментарий), заявка уходит в
отдельный чат лидов (`TELEGRAM_LEAD_CHAT_ID`), управляющее меню от клиентов скрыто.

- Поток: показ товара + `[1][2][3]`/`[✏️ Другое количество]` → (опц. своё число) →
  «Комментарий?/⏭ Пропустить» → «✅ Заявка принята».
- `bot/order_flow.py` (чистая логика, `OrderReply`), `bot/order_dialog.py`
  (состояние в SQLite). Лид: товар, кол-во, комментарий, @клиент, id.
- `publish/orders.py`: `leads` +колонки `qty`/`comment` (ALTER, backward-compat);
  `item_summary` чистит HTML (клиенту/в лид — обычный текст).
- Безопасность: в «разрешённую чужим» зону пропускается строго диалог заказа
  (callback `order:*`, текст-комментарий при активном диалоге) — до owner-гейта.
  Команды (`/…`) опросник не глотает (владелец может выйти).
- `setMyCommands`: управляющее меню — scope владельца; у остальных пусто.

### Дефекты, найденные на red-team-review (закрыты)
1. Подделанный `order:qty:<нечисло>` → `int()` ValueError → падение long-poll.
   Фикс: `_parse_qty` вместо `int(val)`.
2. Команда владельца на шаге комментария превращалась в комментарий (левый лид).
   Фикс: текст на `/` опросник не перехватывает.

## Ops (шаги владельца/деплоя)
- `.env` на проде: `TELEGRAM_LEAD_CHAT_ID=-1004311211079` (иначе лиды — в личку).
- `setMyCommands` применяется при старте `cf-bot` (рестарт после деплоя).
- Живая проверка `<blockquote>` — сначала на тесте/в личку (не в боевой канал).
