# Spec: Волна апгрейда завода (решения владельца 2026-07-07)

**Owner:** Алексей · по итогам разбора 12 идей.

## Решения владельца (дословно важное)

- **№1 Автопилот наполнения — ОБЯЗАТЕЛЬНО.** Товарные ГРУППЫ: КБТ (холодильники,
  морозильные камеры/лари, стиральные машины, телевизоры), МБТ, сплиты, бойлеры.
  «10 КБТ» = разные ДОЛИ разного товара внутри группы (не 10 одинаковых позиций!),
  настраиваемо, разнообразие обязателен.
- **№2 API-генерация картинок — ОТПАДАЕТ**: владелец проверял, gpt-image по
  API-ключам «прям в минус» по качеству против ChatGPT-проектов. НЕ предлагать.
- **№3 Авто-диф прайсов — ДА**, расширено владельцем: **открытые источники**.
  Кнопка «➕ Добавить источник» → «вложите прайс» → файл → следующий шаг:
  «какая наценка?» → владелец пишет ЧИСЛО со знаком (+5 / -7; проценты
  подразумеваются) → источник добавлен и генерится как полноценный. Шкала
  в обе стороны: минус = скидка от прайса, плюс = наценка.
- №4 дайджест, №5 /health, №6 умные цены, №7 приоритеты очереди, №9 /leads-CRM — ДА.
- №10 мультиканальность — отложена («слишком сложно пока»).
- Фидбек по качеству идей: владелец ждёт предложений «на опережение» — следующую
  волну готовить на базе телеметрии (после дайджеста/health будут данные).

## Порядок работ (волнами)

**Волна 1 — фундамент источников и групп:**
1. Динамические источники: prices/sources.json (id, имя, файл, markup_pct, ts);
   legacy-слоты manual/channel/mail остаются встроенными источниками. Наценка
   применяется в ЕДИНОЙ точке — при чтении прайса (load_price_slots), все
   потребители (листинги /find /task, превью) видят уже наценённую цену.
   Глобальный EXCEL_MARKUP_PCT остаётся поверх (сейчас 0) — не складывать
   осознанно без владельца.
2. Бот: /sources — список источников (имя, позиций, наценка, свежесть) +
   кнопки ➕ добавить (файл → наценка), ✏️ наценка, 🗑 убрать.
3. Товарные группы: config groups (КБТ/МБТ/сплиты/бойлеры → список стем-паттернов
   разделов/названий), функция group_of(item). Настройка в config.yaml.

**Волна 2 — автопилот:**
4. Норма наполнения: конфиг «постов в день + доли групп» (напр. 5/день:
   КБТ 2, МБТ 2, сплиты 1). Тик автопилота держит буфер готовых превью:
   если превью в ожидании < нормы завтрашнего дня — добирает из прайсов
   с разнообразием (разные подкатегории/бренды, анти-повтор по истории).
5. Приоритеты: ручные /task-задачи обгоняют автопилотные (колонка priority
   в excel_items; тик берёт ORDER BY priority DESC, ts).

**Волна 3 — наблюдаемость и деньги:**
6. Авто-диф прайсов при обновлении источника: новинки/изменения цен/пропавшие →
   сводка с кнопкой «поставить новинки в работу».
7. Вечерний дайджест (21:00): сгенерено/опубликовано/лидов/ошибок.
8. /health: дорожки (реальный CDP-attach!), таймеры, возраст источников, диск.
9. Умные цены: округление до красивых (25 990), плашка «↓ цена снижена».
10. /leads: статусы (новый/связался/продан), конверсия за неделю.

## Риски
| Риск | Сигнал | Фолбэк |
|---|---|---|
| Двойная наценка (source + глобальная) | цена в превью выше ожидаемой | глобальную держим 0; в /sources показывать итоговую формулу |
| Группы по стемам дают ложные попадания (ТВ-тумба → ТВ) | владелец видит чужой товар в группе | правка паттернов в config.yaml без кода |
| Автопилот съедает лимиты ChatGPT | буфер большой, очередь пухнет | дневной cap генераций в конфиге |
