# Контент-завод (Content Factory)

Автоматический конвейер: **берёт товары из БД → считает цену → пишет краткое описание →
генерирует карточку-картинку (фотоагент) → проверяет правилами → публикует в Telegram-канал**.
Полностью по расписанию/задачам, без ручной возни.

> Это **новый проект**, но не с нуля: «движок» (источник данных, мост к фотоагенту, очередь,
> авто-побудка агента, генерация описаний, режимы карточек, авто-ретрай) **уже построен и обкатан**
> в проекте `avito-bridge` (публикация кондиционеров на Avito). Здесь мы **переиспользуем этот движок**,
> убираем Avito-специфику (XML-фид) и заменяем публикацию на **Telegram-канал**.

## Цель (словами владельца)
Есть «обсервер-агент» (оркестратор), которому ставят задачу: «возьми сегодня 10 карточек из
*<категория>*, сгенерируй краткое описание, отправь в фотоагент, получи карточку, проверь точность,
опубликуй в моём Telegram-канале — завтра в 10:00 и 14:00». Задачи ставятся на день/неделю вперёд,
дальше всё крутится само. Потом владелец добавляет товары/поставщиков/категории — без переписывания.

## Ключевые решения (согласованы с владельцем)
1. **Публикация:** Telegram **Bot API**, бот — администратор канала (`sendPhoto` + подпись). Бот и права уже есть.
2. **Ревизия (точность):** **детерминированные правила, без LLM** (цена>0, ТТХ на месте, карточка
   сгенерирована, описание непустое/без стоп-слов). Не прошло — не публикуем, помечаем.
3. **Постановка задач:** основной путь — **команды боту в Telegram**; плюс поддержать **файлы-планы YAML**
   (оба способа кладут задачи в одну очередь, исполняет планировщик).
4. Контент-движок и описания — **без LLM-ключа** (как в avito-bridge): копирайт детерминированный/шаблонный
   с вариативностью + ручные тексты-переопределения. (LLM можно подключить позже как опцию.)

## Пилот
Первый прогон — на **остатках кондиционеров** (та же БД `oasis`), но публикуем **в Telegram, не в Avito**.
Это даёт рабочий «контент-завод» end-to-end; дальше владелец расширяет категории/поставщиков.
