# CLAUDE.md — рабочее соглашение «Контент-завода»

Ты продолжаешь разработку **Контент-завода** (см. [README.md](README.md)). Это greenfield-проект,
но движок переиспользуется из `../Avito/avito-bridge/`.

## Принципы
- **TDD.** Сначала тест, потом код. `pytest`, держать зелёным. Эталон стиля тестов — avito-bridge/tests.
- **Хирургические изменения.** Трогать только то, что нужно задаче. Не плодить абстракции наперёд.
  Сохранять стиль соседнего кода. Комментарии — по делу, на русском, как в avito-bridge.
- **Без LLM-ключа** в рантайме контента (решение владельца): описания — детерминированные/шаблонные
  с вариативностью + ручные переопределения. LLM — только если владелец явно попросит, как опция.
- **Цель-сначала.** Для каждой задачи — что значит «готово» + конкретная проверка (тест/прогон), и до неё доводить.

## Стек (как в avito-bridge)
Python 3.11+, `pydantic`/`httpx`/`PyYAML`/`python-decouple`/`pytest`. На сервере — `psycopg2`,
Telegram через Bot API (`httpx` к `api.telegram.org`, или `aiogram`/`python-telegram-bot` — на выбор).

## Секреты — НИКОГДА в git
`.env` (gitignored): креды БД, токен фотоагента, **токен Telegram-бота**, id канала. Читать серверные
креды server-side из существующих `.env` (см. HANDOFF), НЕ эхоить в чат. Проверять `git grep` перед коммитом.
⚠️ В avito-bridge `.gitignore` имел широкие `*.txt`/`*.json` — если копируешь, добавь негейты-исключения
для контент-папок (иначе тексты/манифесты не закоммитятся).

## Грабли среды (проверено)
- Шелл — Git Bash на Windows. **Не вкладывать одинарные кавычки в SSH-команду в одинарных кавычках** —
  Python-скрипты для сервера писать в файл и слать через stdin: `ssh … '… python -' < script.py`.
- Команды-цепочки с `&&`: падение локального редиректа (`< нет_файла`) прерывает цепочку до `tar -xzf` —
  проверять, что файл существует.
- Не публиковать/коммитить, пока тесты не зелёные. Отчитываться честно (упал тест — сказать с выводом).
